已修复。关键改动：

## 修复内容

**1. 新增标准工具调用类型定义**
```typescript
interface StandardToolCall {
  id: string;
  type: "function";
  function: {
    name: string;
    arguments: string;
  };
}
```

**2. 修复 `tool_calls` 类型断言**
```typescript
// 之前：直接遍历，TS 报错
for (const toolCall of responseMessage.tool_calls) {
  toolCall.function.arguments  // ❌ TS 错误
}

// 现在：双重断言绕过 SDK 类型定义 bug
const toolCalls = responseMessage.tool_calls as unknown as StandardToolCall[];
for (const toolCall of toolCalls) {
  toolCall.function.arguments  // ✅ 正常
}
```

## 问题根因

OpenAI SDK v6 的 `ChatCompletionMessageCustomToolCall` 类型定义不完整，缺少 `.function` 属性。使用 `as unknown as StandardToolCall[]` 双重断言可以安全地绕过这个 SDK 自身的类型 bug。
